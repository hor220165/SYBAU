# Beitrag-Regeln (CONTRIBUTING)

Bevor man irgendetwas ändert immer zuerst pullen um den neusten Stand zu holen

## Workflow
1. git pull:
   `git pull`
2. Änderungen machen
3. git add:
  `git add .` 
4. Commit:
   `git commit -m "<änderung>"`
5. Push:
   `git push`

## Commit-Namen
- `feat: ...` – neues Feature
- `fix: ...` – Bugfix
- `docs: ...` – Dokumentation
