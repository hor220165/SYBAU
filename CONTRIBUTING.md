# Beitrag-Regeln (CONTRIBUTING)

## Branches
- `main` – stabile Version
- `dev` – aktuelle Entwicklung
- `feature/<name>` – neue Features
- `bugfix/<name>` – Fehlerbehebungen

## Workflow
1. Neuen Branch erstellen:
   `git checkout -b feature/<name>`
2. Änderungen machen
3. Commit:
   `git commit -m "feat: <änderung>"`
4. Push:
   `git push origin feature/<name>`
5. Pull Request auf GitHub öffnen
6. Eine andere Person reviewed & merged

## Commit-Namen
- `feat: ...` – neues Feature
- `fix: ...` – Bugfix
- `docs: ...` – Dokumentation
- `style: ...` – Formatierungen
